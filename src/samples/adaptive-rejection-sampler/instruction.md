A colleague implemented an adaptive rejection sampler (Gilks & Wild 1992) in R, and I want to grade submissions of that task automatically. Each attempt writes `/app/ars.R` and generates a sample file. Write a verifier that decides whether a given attempt is a correct implementation. R is available, and the attempt's files are the only thing to judge — source and run `ars.R` to see how it behaves.

A correct `ars.R` defines `ars(g, D, n)` that returns `n` draws from the density proportional to `g` (a vectorized density function such as `dnorm` or `dexp`) over the domain `D = c(lower, upper)`. It must work for any log-concave `g` — a shifted or scaled normal, an exponential, and so on — not only the standard normal, and the draws must follow the target distribution's shape, not merely land near its mean. It must reject invalid inputs (a non-positive `n`, or a domain whose bounds are not increasing) and non-log-concave densities (for example a bimodal mixture) by raising an error. It also defines a `test` function and writes `normal_samples.txt` or `exponential_samples.txt`. Sampling is stochastic, so account for that. Judge the behavior, not the coding style or which helper functions an attempt happens to define; extra functions or files do not make an attempt wrong.

Write your verifier as an executable at `/app/verifier/verify.sh`. It is run once per attempt: it reads the attempt's files from `SOLUTION_DIR` (default `/app`), may read agent logs from `ARTIFACTS_DIR`, and writes `1` (correct) or `0` (incorrect) to `VERDICT_FILE` (default `/logs/verifier/reward.txt`).
